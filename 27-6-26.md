## How to read this board

Every item is grouped **by page** and tagged:

- **Type** — 🐛 Bug (broken existing behavior) · 🎨 Tweak (visual: spacing/color/font) · ✨ Feature (new capability) · ❓ Decide (open question)
- **Owner** — `FE` (mine) · `BE` (teammate) · `FE+BE` (needs both — FE ships a **placeholder/mock** first)
- **Status** — ❌ Open · 🔶 Partial · ✅ Done · ❓ Unverified
- **Model** — suggested Claude model: `Haiku` (color/copy/small fix) · `Sonnet` (most features/bugs) · `Opus` (planning/architecture)

**Scheduling rule:** items in the same page that touch the same file run **sequentially**; items
across different pages with no shared file can run **in parallel**. Backend items are dispatched to
the teammate — I keep a `const MOCK = true` flag + placeholder UI so frontend never blocks on backend.

# Appendix A — Workflow & token discipline

_From `mental.md` + `plan.md`. Usage stats showed 97% subagent-heavy, 74% sessions 8h+, 32% >150k context._

**Per-task prompt template:**
```plain
[What I'm trying to do]:  add a top to my wardrobe
[What happens]:           Add Item → modal opens → I upload a photo
[What feels wrong]:       modal too big, confirm button hidden below fold
[What I want instead]:    fits without scrolling / button always visible
```

**Scoping frontend-only work:** if it lives in `frontend/`, it's FE. When an API isn't ready, mock it:
```ts
const MOCK = true; // flip when backend is ready
const data = MOCK ? mockWardrobeItems : await api.get('/wardrobe');
```

**Model selection:** Haiku = color/copy/rename · Sonnet = 90% of coding · Opus = planning/architecture.

**Subagents:** only when you *can't point to the file*. Know the file → inline edit. Don't know → cheap read-only `Explore` agent.

**Session hygiene:** `/clear` when switching feature · `/compact` after 3–4 tool calls or 2 failed attempts · `/commit` after every working state · `/verify` after a UI change. Target: ≤2–3 tasks per session.

**Decision tree:** know file? → inline. Don't? → Explore. Context long? → `/compact`. New task? → `/clear`. 2h+? → commit, clear, restart.

# Appendix C — Design vocabulary (quick reference)

**Feel words → direction:** Premium = more whitespace, less color, serif, subtle · Clean = high contrast,
geometric sans, minimal · Warm = rounded, warm neutrals, loose spacing · Editorial = mixed type sizes,
asymmetry, grid-breaking. **StyleSense = editorial + warm** (EB Garamond, earth tones).

**The 5 questions** when something feels off: is it a **space** (whitespace/padding/gap), **size** (font/dimensions),
**color** (contrast/saturation/value), **hierarchy** (important thing doesn't look important), or **feel**
(right elements, wrong mood) problem? Name one → that's a complete brief.
